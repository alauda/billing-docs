# Основные концепции

## Opencost

**OpenCost** — это инструмент для мониторинга и управления затратами в Kubernetes с открытым исходным кодом, предназначенный для отслеживания и анализа использования ресурсов и затрат в Kubernetes кластерах в реальном времени. Он предоставляет точные данные о замере ресурсов, чтобы помочь пользователям оптимизировать распределение ресурсов и контролировать затраты.

Для получения дополнительной информации посетите [официальный сайт OpenCost](https://www.opencost.io/docs/).

## Описание методов биллинга

| Пункт биллинга | Метод биллинга              | Правила биллинга                                                                                                                                                                                                                                                                           | Описание                                                                                                             |
| --------------- | --------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------- |
| **CPU**        | Эффективное использование (Ядро-часа) | Рассчитывается на почасовой основе с использованием большего значения среднего потребления (AVG(Usage)) и запрашиваемого объема (Request) POD за последний час, умноженного на фактическую продолжительность работы POD (исчисляется как 5 минут, если менее 5 минут).   |                                                                                                                         |
| **Память**     | Эффективное использование (ГиБ-часа)   | Рассчитывается на почасовой основе с использованием большего значения среднего потребления (AVG(Usage)) и запрашиваемого объема (Request) POD за последний час, умноженного на фактическую продолжительность работы POD (исчисляется как 5 минут, если менее 5 минут).   |                                                                                                                         |
| **Хранилище**  | Запрашиваемый объем (ГиБ-часа)     | Рассчитывается на почасовой основе с использованием размера запроса (Request) PV за последний час, умноженного на фактическую продолжительность работы PV (исчисляется как 5 минут, если менее 5 минут). Если PV одновременно смонтирован к нескольким POD, использование PV будет равномерно распределено между ними. | PV, которые не смонтированы ни к одному POD, будут учитываться как не смонтированные записи, при этом имя объекта будет `*unmounted-pvcs`. |

Текущие статистические проблемы будут решены в будущих версиях:

- Когда POD StatefulSet останавливается и перезапускается, платформа будет использовать самое раннее время работы POD в тот день в качестве стартового времени и самое позднее время работы в качестве конечного времени, игнорируя при этом время, в течение которого POD не работал.
