# 介绍

在快速发展的云计算时代,企业需要一种灵活、精准的计费模式来管理和优化云资源开支。平台提供了一款专为云原生场景打造的计量计费产品，帮助您实现精细化的资源管理和成本控制。

平台提供了通用的计量计费能力及基础的计费方式，支持对集群内的 CPU、内存、存储资源进行计量计费，并提供统计、查看明细等能力。

- 费用统计:支持平台管理员对各租户的费用进行汇总统计。
- 费用明细:支持平台管理员对各租户的费用明细进行查询，以便对账。


## 优势
- **精细化资源管理**：精确计量 CPU、内存、存储资源的使用情况，帮助用户深入了解资源消耗。
- **成本控制**：通过详细的计量数据和计费信息，用户可以更好地控制和优化资源成本。
- **灵活的计费方式**：支持多种基础计费方式，满足不同用户的需求。
- **透明化**：提供资源使用统计和明细查看功能，增强资源使用的透明度。
- **集成 OpenCost**：基于 OpenCost，确保计量数据的准确性和可靠性。

## 应用场景
- **企业资源管理**：帮助企业精确计量和分摊 IT 资源成本，优化资源分配。
- **云原生应用**：适用于云原生环境下的微服务架构，精确计量每个服务的资源消耗。
- **多租户环境**：在多租户环境中，为每个租户提供独立的资源计量和计费，确保公平性和透明度。
- **成本优化**：通过详细的资源使用数据，识别和优化高成本资源，降低总体运营成本。

## 使用限制
- **依赖 Prometheus**：计量数据基于 Prometheus 获取，安装前请确保需要计量的集群已经部署了 Monitor with Prometheus 插件。
- **资源类型限制**：目前仅支持 CPU、内存和存储资源的计量计费，其他资源类型暂不支持。
- **集群兼容性**：功能可能不兼容所有类型的 Kubernetes 集群，需在支持的集群环境中使用。
- **数据延迟**：由于数据采集链路存在延时性，计量数据可能存在一定的延迟。

基于此功能，我们希望能够帮助用户更好地管理和优化其容器平台的资源使用，实现更高效的成本控制。
